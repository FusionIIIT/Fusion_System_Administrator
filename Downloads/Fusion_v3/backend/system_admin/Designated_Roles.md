# Module Name: System Admin

## Designated User Roles & Permissions

### 1. Role Name: Super Admin

* **Description:** Primary manager for System Admin module settings, user lifecycle, and RBAC.
* **Permissions:**
    * Full CRUD operations on users, roles, and module permissions.
    * Assign, reassign, and revoke designations and module access.
    * Access audit logs, emergency access approvals, and archival actions.

### 2. Role Name: Faculty/Staff

* **Description:** Operational user supporting user management tasks within assigned scope.
* **Permissions:**
    * View assigned users and role mappings.
    * Edit allowed fields (status updates, profile details) within scope.
    * Generate summaries and reports for assigned users.

### 3. Role Name: Student/End-User

* **Description:** Consumer of System Admin services for personal account data.
* **Permissions:**
    * Read-only access to personal records and role status.
    * Submit requests (account updates or access requests) via the frontend.
